Словарик

Repository - хранилище для всех версий кода

Полезные команды:

  • git clone - позволяет стянуть репозиторий себе на комп
  • git pull - позволяет стянуть все изменения, произошедшие удаленно, себе на комп
  • git checkout - позволяет переключаться между ветками (пример git checout dev, где dev - название ветки)
  • git add - позволяет сделать так, чтобы git увидел новый файл, подготавливаем изменения к коммиту (пример git add . - добавляет все файлы, git add newFile - добавляет файл newFile)
  • git commit -m '' - позволяет сделать так, чтобы git посчитал добавленный файл базовой версией кода (пример git commit -m 'add newFile' - в кавычках указывается message)
  • git status - позволяет посмотреть в каком статусе находится проект
  • git push - позволяет отправить изменения с локального компа на удаленный сервер
  • pull request - позволяет добавлять изменения из ветки, где разрабатывалась новая фича, в ветку master

В каких статусах может находиться файл с точки зрения git?

  1. untracked status - когда добавили файл в проект, а git про него еще ничего не знает
  2. git add - команда для того, чтобы git стал отслеживать новый файл, чтобы его можно было закоммитить
  3. git commit -m '' - команда для того, чтобы git отслеживаемый файл стал считать базовой версией кода
  4. git push - команда для того, чтобы отправить изменения с локального компа на удаленный сервер
  5. pull request - команда для того, чтобы добавить изменения из ветки, где проводились изменения, в ветку master (главную ветку)

Что еще может быть полезно:

  • Github Pages - публикация кода из репозитория в интернете (как найти: в репозитории settings > pages)

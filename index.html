Словарик

Repository - хранилище для всех версий кода

Полезные команды:

  • git init - позволяет создать репозиторий
  • git clone - позволяет стянуть репозиторий себе на комп
  • git pull - позволяет стянуть все изменения, произошедшие удаленно, себе на комп
  • git checkout - позволяет переключаться между ветками (пример git checout dev, где dev - название ветки)
  • git add - позволяет сделать так, чтобы git увидел новый файл, подготавливаем изменения к коммиту (пример git add . - добавляет все файлы, git add newFile - добавляет файл newFile)
  • git commit -m '_' - позволяет сделать так, чтобы git посчитал добавленный файл базовой версией кода (пример - в кавычках указывается message)
  • git status - позволяет посмотреть в каком статусе находится проект
  • git push - позволяет отправить изменения с локального компа на удаленный сервер
  • pull request - позволяет добавлять изменения из ветки, где разрабатывалась новая фича, в ветку master
  • rm - позволяет удалить файл, но после удаления нужно использовать git add и git commit (пример rm fileForRemove)
  • git restore - позволяет восстановить удаленный файл (пример git restore deletedFile)
  • git rm - позволяет удалить файл, но после удаления нужно использовать только git сommit (равносильно rm + git add, то есть команда объединяет в себе удаление и подготовку к коммиту)
  • git diff - позволяет увидеть разницу между изменением, что было и что стало, но до того, как использовалась команда git add
  • git diff --staged - позволяет увидеть разницу между изменением, что было и что стало, но после того, как использовалась команда git add, то есть после того, как было подготовлено к коммиту
  • git mv - позволяет переименовать файл (пример git mv oldFile newFile)
  • git log - показывает список всех выполненных коммитов, отсортированных по дате добавления
  • git log -p - показывает коммиты с полными дифами, изменениями
  • git show - показывает диф между этим комитом и предыдущем (пример git show 5120bea3e5528c29f8d1da43731cbe895892eb6d либо git show 5120bea – идентификатор-хеш, первые семь цифр уникальны, можно их использовать)
  • git blame - выводит файл и рядом со строчкой показывает, кто ее менял и в каком комите (пример git blame _путь_до_файла_ )
  • git grep - находит совпадение с указанной строкой во всех файлах проекта (пример git grep _слово_ )
  • git grep -i - находит совпадение без учета регистра (пример git grep -i _слово_ )
  • mkdir - создает пустую директорию (пример mkdir _название_файла_ )
  • touch - открывает файл (пример touch _название_файла_ )
  • git clean -fd - удаляет -f – forced, -d – directory

В каких статусах может находиться файл с точки зрения git?

  1. untracked status - когда добавили файл в проект, а git про него еще ничего не знает
  2. git add - команда для того, чтобы git стал отслеживать новый файл, чтобы его можно было закоммитить
  3. git commit -m '_' - команда для того, чтобы git отслеживаемый файл стал считать базовой версией кода
  4. git push - команда для того, чтобы отправить изменения с локального компа на удаленный сервер
  5. pull request - команда для того, чтобы добавить изменения из ветки, где проводились изменения, в ветку master (главную ветку)
